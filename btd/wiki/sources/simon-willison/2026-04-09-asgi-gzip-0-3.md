# asgi-gzip 0.3

Bugfix release: asgi-gzip was incorrectly compressing Server-Sent Events (`text/event-stream`) responses, breaking SSE in production Datasette deployments. Fix was available in Starlette but a scheduled GitHub Actions workflow had silently stopped running and missed it. Practical lesson in maintaining automated upstream-tracking workflows.

## Key Ideas
- SSE responses must not be buffered/compressed — gzip middleware needs explicit exemptions for `text/event-stream`
- Scheduled GitHub Actions workflows can stop silently; periodic manual checks or alerting on workflow failure are necessary
- Extracting shared code into a library creates a maintenance obligation to track the upstream source for fixes
