# datasette-ports 0.1

A new Datasette plugin (built via README-driven development) that lists all running Datasette instances with their ports, databases, and plugins. Solves a real friction point when running many Datasette instances across development sessions. Another example of using AI to ship a polished, well-documented tool for a very specific personal pain point.

## Key Ideas
- README-driven development: write the complete spec and README first, then hand to Claude Code to implement — the result is a tool that ships with documentation already done
- Niche tools for personal workflow friction are now worth building; the cost-benefit calculation changed when AI handles implementation
- Datasette's plugin ecosystem makes it easy to extend with targeted utilities
