# Claude Code Turned Obsidian Into My Dream Second Brain

Practical setup guide for using Claude Code with Obsidian's CLI to build a persistent, AI-navigable second brain. Kashef walks through installing Obsidian's 95-command CLI, activating it in settings, layering on custom skills (Obsidian CLI skill + JSON canvas skill), and using a `vault setup` slash command to configure a personal vault via multiple-choice questions. The key workflow: convert noisy PDFs/docs to markdown via Gemini's large context window, compress to cheat sheets, then ingest into Obsidian via the CLI. Useful for builders who want Claude Code to have persistent memory across sessions.

## Key Ideas
- Obsidian's CLI exposes 95 commands; Claude Code can operate all of them via natural language
- Skills bridge Claude Code to Obsidian: one for CLI control, one for JSON canvas diagrams
- Vault setup skill interviews you (work/personal, what falls through cracks, existing files) to scaffold your folder structure
- PDF-to-second-brain pipeline: Claude Code organizes files → Gemini compresses PDFs to markdown → Obsidian CLI ingests
- Claude Code sessions can open directly in the Obsidian vault folder for full context injection
