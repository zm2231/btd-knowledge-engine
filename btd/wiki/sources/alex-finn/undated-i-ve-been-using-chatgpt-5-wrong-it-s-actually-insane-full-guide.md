# I've been using ChatGPT-5 wrong. It's actually insane (full guide)

Finn's full guide to getting dramatically better results from GPT-5, focusing on underused capabilities: deep research mode, multi-modal inputs, canvas for iterative document work, and agent tasks. The framing is a correction — most users treat it like a search engine rather than a capable reasoning partner. Useful BTD context for builders comparing GPT-5 and Claude workflows.

## Key Ideas
- Deep research mode fundamentally changes GPT-5 — it browses, synthesizes, and cites sources autonomously
- Canvas (iterative document editor) is GPT-5's equivalent of Claude's artifact workflow — high value for writing tasks
- Multi-modal: uploading images, PDFs, and spreadsheets alongside text prompts unlocks much richer analysis
- Agent tasks let GPT-5 run multi-step workflows asynchronously — similar to Claude Code's approach
- Most GPT-5 underuse comes from chat-box habits; the real power is in projects, memory, and tool use
