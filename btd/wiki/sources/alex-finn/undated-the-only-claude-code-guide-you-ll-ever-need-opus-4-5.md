# The only Claude Code guide you'll ever need (Opus 4.5)

Comprehensive Claude Code guide updated for Opus 4.5, covering setup, CLAUDE.md configuration, skills, MCP servers, and real-world workflows. Finn's strongest single reference for anyone serious about Claude Code. Includes model-specific guidance on when to use Opus vs. Sonnet vs. Haiku within a Claude Code session for cost/quality balance.

## Key Ideas
- Opus 4.5 handles complex multi-step reasoning significantly better than Sonnet for autonomous long-horizon tasks
- CLAUDE.md structure: project context, your preferences, current goals, and any constraints — keep it under 500 tokens
- Skills are the multiplier: invest time designing two or three core skills before trying to build elaborate multi-agent systems
- MCP servers extend Claude Code's tool set; use only what you need — each server adds latency and context overhead
- Cost management: use Haiku for reads/classifies, Sonnet for drafts, Opus only for decisions requiring deep reasoning
