# This week on How I AI: I gave Claude Code our entire codebase. Our customers noticed.

Newsletter announcement post for the How I AI podcast episode featuring Al Chen (Galileo). Claire Vo and Al Chen discuss how Al uses Claude Code to query Galileo's 15 repositories to answer enterprise customer questions in real time without documentation or engineering support. The post summarizes the five biggest takeaways from the episode, making it a quick-read entry point to the full episode for Lenny's Substack audience. Content substantially overlaps with the main article post; this version is formatted as a promotional teaser with the episode's biggest practical lessons upfront.

## Key Ideas
- Write a daily sync script to pull the latest code from all repos — let Claude Code write the script itself
- Code is more reliable than documentation as a source of truth for technical support
- MCPs let you combine multiple knowledge sources (repos + Confluence + Slack) into a single coherent context
- Turn each customer question into a reusable knowledge artifact, not just a one-off answer
- Non-engineers can now operate as technical support through AI tooling — the skills gap is shrinking
