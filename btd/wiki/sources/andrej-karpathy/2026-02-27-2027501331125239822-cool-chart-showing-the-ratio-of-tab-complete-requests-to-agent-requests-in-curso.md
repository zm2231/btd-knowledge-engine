# The evolving optimal AI coding workflow

Using a Cursor chart showing the rising ratio of agent to tab-complete requests over time, Karpathy makes a broader point: as model capability increases, the optimal level of AI delegation keeps shifting — and you have to track it. Being too conservative (still using just autocomplete) leaves leverage unrealized; being too aggressive (letting agents run unchecked) creates more chaos than productivity.

## Key Ideas
- The optimal AI coding setup is not stable — it tracks model capability, which is improving quarterly
- The evolution: no AI → autocomplete → agents → parallel agents → agent teams — each step requires intentional recalibration
- Under-delegation and over-delegation are symmetric failure modes; the art is staying near the current frontier
- Cursor usage data shows the community does track and update its practices, which is a useful signal for individual recalibration
