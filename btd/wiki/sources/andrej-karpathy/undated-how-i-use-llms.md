# How I use LLMs

A practical 2-hour walkthrough of how Karpathy actually uses ChatGPT, Claude, Gemini, and other LLM products in his day-to-day work — not as a demo reel but as a workflow audit. Covers model selection, what tasks each product handles well, how to structure prompts for reliable results, and the failure modes to watch for. The companion piece to his "Deep Dive" video: where that one explains how LLMs work, this one shows how to use them effectively right now.

## Key Ideas
- Different models have genuine capability differences for different task types — model selection is a real decision, not cargo-culting
- LLMs are best used as "thought partners" — externalize your reasoning, get pushback, iterate — not as answer machines to interrogate
- Keeping a personal "prompt library" of working patterns dramatically raises reliability; good prompts are reusable infrastructure
- The most common failure mode is under-specifying the task — the model will give you an answer, but not necessarily the answer to the question you meant to ask
- Voice interfaces change the use pattern significantly: faster for exploratory thinking, worse for precision tasks that need structured output
