# The IDE evolves: programming moves up a level of abstraction

Pushback on the narrative that "IDEs are dead because AI agents write code." Karpathy's take: IDEs aren't disappearing, they're growing. Humans are moving up the abstraction ladder — the basic unit of interest shifts from one file to one agent — but it's still programming. The IDE of the future manages agents, not files.

## Key Ideas
- The IDE isn't dead; the scope of what it manages expands to include agent orchestration, not just code editing
- Programming continues — humans just operate at a higher level of abstraction, specifying agent goals rather than implementation details
- "One agent" becomes the new atomic unit the way "one file" used to be — and IDEs will evolve to reflect this
- The patterns that work for file-level programming (version control, testing, review) need analogues at the agent level
