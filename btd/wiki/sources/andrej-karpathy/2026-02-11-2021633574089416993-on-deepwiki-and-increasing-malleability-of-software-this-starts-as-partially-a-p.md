# DeepWiki and the malleability of software

Karpathy uses DeepWiki (a tool that auto-builds wiki pages for any GitHub repo) as a jumping-off point to explore a larger thesis: software is becoming more malleable because AI agents can read, understand, and modify code that humans would struggle to navigate. DeepWiki swapping "github" for "deepwiki" in a URL to get instant documentation is a small example of a bigger pattern — the barrier between "understanding a codebase" and "using a codebase" is collapsing.

## Key Ideas
- DeepWiki generates browsable documentation + Q&A for any public GitHub repo by swapping the URL prefix — trivially low friction
- The deeper point: as AI reads and writes code fluently, the "learning curve" for any codebase approaches zero
- Software malleability means that user interfaces, integrations, and customizations that used to require developers can be generated on demand
- This changes the economics of software — the bottleneck shifts from "can we build this?" to "what should we build?"
