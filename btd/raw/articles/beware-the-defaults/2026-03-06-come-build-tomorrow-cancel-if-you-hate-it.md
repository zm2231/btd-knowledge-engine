Starting Friday (tomorrow 3/6), I’m running a 3-day Vibe Coding sprint inside the [BTD community.](https://www.skool.com/bewarethedefault/about) ​

Three sessions. You walk away with a finished app, even if you have zero idea what to build right now.

I ran the first version of this a few months ago. Learned a ton from it and now it’s time to run it back.

**Here’s the schedule (booking it for 60 minutes but may run over):**

- Day 1: Friday, during the normal Build Lab slot

- Day 2: Tuesday 3/10, 3pm EST

- Day 3: Thursday, 3/12 3pm EST


Everything is recorded. I’ll open a thread each day so you can drop questions and stuck points as you build, + have a day in between sessions to do some deep-vibin-work.

​

There are also three other people  that run the community that are much smarter than me, so there will be no shortage of help.

## **What we’re actually covering:**

**Starting from scratch on what a good idea is.** How to pick one, how to know if it’s real, how to decide if people would pay for it or if it’s just an “annoying” problem, how to position it, how to turn it into something you can ship in a week or two.

This is not JUST a Lovable tutorial. If you know Lovable, great. If you don’t, you’ll be fine. The tools will probably change next week. **I want to teach you actual strategy.** Skills that you can use no matter what these crazy AI people in fancy offices come up with.

I also built a vibe coding buddy to use during the sprint (his name is Rubber Ducky Robot - it will make more sense once you get in there). It will hold your hand the entire way from idea to finished product no matter how technical you are. It will _literally_ will give you every prompt to give Lovable.
